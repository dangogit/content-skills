# CTA Trust Check

Run CTA and delivery flow through this before shipping.

## Fail Conditions

- [ ] Promise is vague, unnamed, or unrelated to post.
- [ ] Link or DM delivers sales page before promised resource.
- [ ] Keyword collides with another active route.
- [ ] Resource URL, route, health, or synthetic delivery proof is missing.
- [ ] CTA stacks several primary asks.
- [ ] Comment request has no content, resource, opinion, or useful next step.
- [ ] Claim depends on fake urgency, fake scarcity, or invented proof.

## Pass Conditions

- [ ] One primary action matches objective.
- [ ] Named artifact is specific and useful.
- [ ] Viewer knows what arrives after action.
- [ ] Promise and delivery match exactly.
- [ ] Keyword route is active and collision-safe when used.
- [ ] Measurement plan exists.

## Rewrites

| Weak | Better |
|---|---|
| "Comment YES for link" | "כתבו ROUTE ואשלח לכם את חוק הברזל למודל" |
| "Tag three friends" | "שלחו את זה למי שבונה עכשיו סוכן ראשון" |
| "Follow for more" | Remove, or name specific series benefit |
| "Save this" | Name future moment when viewer needs it |

Keyword CTA is not automatic failure. Broken promise is failure.
